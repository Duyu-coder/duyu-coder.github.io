浅析编译流程及编译器的实现

1.词法分析

理论

本质是将字符流转换成token流，是一个划分token并整理输出词法表的过程，也为全局的符号表起了个头，后续可以补上作用域等。可以拓展行号追踪、报错等，需要实现跳过空白字符。

逐字符的匹配需要频繁回退，所以使用有限自动机优化。

实现

Flex能自动实现自动机，且通过%% 分为三部分，逐步完善这三部分即可：

1.定义段：给出正规集，如DIGIT、INT_NUM等基本字符类，通过正则表达式完成定义，如：

INT_NUM	[1-9]{DIGIT}*|0
//其中DIGIT [0-9]

2.规则段：由正则模式+C动作组成，如：

{SPACE}	{/*跳过空白字符*/}
"hang"	{return TOK_HANG;}

3.用户代码段，按需处理yywrap等函数，由于与bison等联动，主函数不在此处且不需要其他函数实现，所以此处可以为空。

代码

%{
#include "common.h"
#include "syntax.tab.h"

// 不再需要定义 symbolTable 和 symbolCount

"}"       { return TOK_RH; }  

.         {

    return TOK_ERROR;

与语法部分的联动

词法分析器和语法分析器在代码层面的联动，实质上是通过整数编号达成一致的。

上面定义了很多’TOK_’，这些是符号名，Bison会自动为它们分配唯一的整数编号，也可以自行确定从某个数

开始（如255），这样就完成了词法表的基本要求，即类似下面表格：

序号 类型码 类型值 助记
1 1 - $IF
2 2 - $LS
3 3 $ret $ID
4 4 - $DE
5 5 $2 $ID
6 6 - $RS
……

2.语法分析

理论

本质是接受上面词法分析器输出的token流，按照预先定义的语法规则（产生式）判断是否符合语法（是不是句子），匹配后执行相应的动作。

实现

同样是完成三个部分：
1.声明段：需要完成头文件的引入、语法规则的实现（那就需要终结符/非终结符以及由此组合得到的语法规则；特别的，运算中需要考虑结合性和优先级），并给出匹配后的具体操作。

引入包含全局数据结构（符号表、四元式等）的common.h，声明词法、语法分析器并增加其他想要增加的拓展。

定义语义值类型联合体%union,返回char *或者ASTNode *,以便词法分析器可以通过char *传递字符串（如常量、标识符名）。并声明终结符和非终结符的类型，如：

%token <string_value> TOK_INT_TYPE TOKE TOK_BOOL_TYPE 	//终结符示例
%type <node> program statements statement				//非终结符示例

优先级和结合性方面，后声明的优先级更高且可以显式说明结合性，还可以通过%nonassoc使之不能连续出现，比如括号内不能为空，具体示例如下：

%left TOK_ADD TOK_SUB
%left TOK_MUL TOK_DIV
%nonassoc TOK_GT TOK_LT TOK_GE TOK_LE TOK_EQ TOK_NE

2.规则段（核心）

参照c++和该语言的格式即可，详细代码在代码部分了，这里不多解释，只举一个例子：

statement:
    declaration_statement { $$ = $1; }
    | assignment_statement { $$ = $1; }
    | input_statement     { $$ = $1; }
    | output_statement    { $$ = $1; }
    | when_statement      { $$ = $1; }
    | expression TOK_FEN  { $$ = $1; }
    ;

3.用户代码段

  1. 全局变量
ASTNode* ast_root = NULL;

· 存储整个语法树根节点，供后续语义分析和代码生成使用。

  1. 错误处理 yyerror
void yyerror(const char* s) 
{
    extern int yylineno;
    fprintf(stderr, "语法错误 (行%d): %s\n", yylineno, s);
}

· yylineno 是 Flex 提供的全局变量（需在 Flex 中开启 %option yylineno），用于输出错误行号。

  1. main 函数（部分）
int main(int argc, char* argv[]) 
{
    if (argc > 1) {
        yyin = fopen(argv[1], "r");
        ...
    }
    // 开始计时
    start_timer(&compile_timing.lex_start);
    start_timer(&compile_timing.parse_start);
    int result = yyparse();   // 调用 Bison 生成的语法分析器
    record_parse_time();

· 打开源文件，设置 yyin。
· yyparse() 内部反复调用 yylex() 获取 token 并执行语法分析，最终返回 0 表示成功。
· 计时并记录性能数据。

%{
#include "common.h"
#include "optimize.h"
// 声明词法分析器函数
extern int yylex(void);
extern int yyparse(void);
extern FILE* yyin;

// 错误处理函数声明
void yyerror(const char* s);

// 语法树根节点声明
extern ASTNode* ast_root;
%}

// 定义语义值类型
%union {
    char* string_value;
    ASTNode* node;
}

// 为token和nonterminal指定语义值类型
%token <string_value> TOK_ID TOK_INT TOK_FLOAT TOK_TRUE TOK_FALSE
%token <string_value> TOK_INT_TYPE TOK_FLOAT_TYPE TOK_BOOL_TYPE
%token TOK_HANG TOK_IN TOK_OUT TOK_WHEN
%token TOK_ADD TOK_SUB TOK_MUL TOK_DIV
%token TOK_ASSIGN TOK_FEN TOK_DOU TOK_LY TOK_RY TOK_LH TOK_RH
%token TOK_ERROR

// 添加比较运算符token
%token TOK_GT TOK_LT TOK_GE TOK_LE TOK_EQ TOK_NE

// 定义运算符优先级和结合性
%left TOK_ADD TOK_SUB
%left TOK_MUL TOK_DIV
%nonassoc TOK_GT TOK_LT TOK_GE TOK_LE TOK_EQ TOK_NE

// 为non-terminal指定类型
%type <node> program statements statement
%type <node> declaration_statement assignment_statement input_statement output_statement
%type <node> when_statement type_specifier identifiers expression condition

%%

// 程序由hang块组成 - 压缩hang_block中间节点
program:
    TOK_HANG TOK_LH statements TOK_RH
    {
        $$ = createNode(NODE_PROGRAM, "hang_program");
        // 将语句链表的所有节点添加为子节点
        ASTNode* current = $3;
        while (current != NULL) 
        {
            ASTNode* next = current->next_sibling;
            current->next_sibling = NULL; // 清除兄弟指针，转换为父子关系
            addChild($$, current);
            current = next;
        }
        ast_root = $$;
        printf("语法分析: 识别到hang程序块\n");
    }
    ;

// 语句序列
statements:
    statement
    {
        $$ = $1;
    }
    | statements statement
    {
        // 将新语句添加为兄弟节点
        addSibling($1, $2);
        $$ = $1;
    }
    ;

// 语句定义
statement:
    declaration_statement
    {
        $$ = $1;
    }
    | assignment_statement
    {
        $$ = $1;
    }
    | input_statement
    {
        $$ = $1;
    }
    | output_statement
    {
        $$ = $1;
    }
    | when_statement
    {
        $$ = $1;
    }
    | expression TOK_FEN
    {
        $$ = $1;
    }
    ;

// 声明语句
declaration_statement:
    type_specifier identifiers TOK_FEN
    {
        $$ = createNode(NODE_DECLARATION, "decl");
        addChild($$, $1);  // 类型
        addChild($$, $2);  // 标识符列表
    }
    | type_specifier TOK_ID TOK_ASSIGN expression TOK_FEN
    {
        $$ = createNode(NODE_DECLARATION, "decl_init");
        addChild($$, $1);  // 类型
        ASTNode* id_node = createNode(NODE_IDENTIFIER, $2);
        addChild($$, id_node);
        addChild($$, $4);  // 初始值表达式
    }
    ;

// 类型说明符
type_specifier:
    TOK_INT_TYPE
    {
        $$ = createNode(NODE_TYPE, "int");
    }
    | TOK_FLOAT_TYPE
    {
        $$ = createNode(NODE_TYPE, "float");
    }
    | TOK_BOOL_TYPE
    {
        $$ = createNode(NODE_TYPE, "bool");
    }
    ;

// 标识符列表
identifiers:
    TOK_ID
    {
        $$ = createNode(NODE_IDENTIFIER, $1);
    }
    | identifiers TOK_DOU TOK_ID
    {
        // 将新标识符添加为兄弟节点
        addSibling($1, createNode(NODE_IDENTIFIER, $3));
        $$ = $1;
    }
    ;

// 赋值语句
assignment_statement:
    TOK_ID TOK_ASSIGN expression TOK_FEN
    {
        $$ = createNode(NODE_ASSIGNMENT, "=");
        addChild($$, createNode(NODE_IDENTIFIER, $1));
        addChild($$, $3);
    }
    ;

// 输入语句 - 只接受标识符
input_statement:
    TOK_IN TOK_LY TOK_ID TOK_RY TOK_FEN
    {
        $$ = createNode(NODE_INPUT, "in");
        addChild($$, createNode(NODE_IDENTIFIER, $3));
    }
    ;

// 输出语句
output_statement:
    TOK_OUT TOK_LY expression TOK_RY TOK_FEN
    {
        $$ = createNode(NODE_OUTPUT, "out");
        addChild($$, $3);
    }
    ;

// when语句
when_statement:
    TOK_WHEN TOK_LY condition TOK_RY TOK_LH statements TOK_RH
    {
        $$ = createNode(NODE_WHEN, "when");
        addChild($$, $3);  // 条件
        addChild($$, $6);  // 语句块
    }
    ;

// 条件表达式 - 修复规约冲突
condition:
    expression
    {
        $$ = $1;
    }
    | expression TOK_GT expression
    {
        $$ = createNode(NODE_OPERATOR, ">");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_LT expression
    {
        $$ = createNode(NODE_OPERATOR, "<");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_GE expression
    {
        $$ = createNode(NODE_OPERATOR, ">=");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_LE expression
    {
        $$ = createNode(NODE_OPERATOR, "<=");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_EQ expression
    {
        $$ = createNode(NODE_OPERATOR, "==");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_NE expression
    {
        $$ = createNode(NODE_OPERATOR, "!=");
        addChild($$, $1);
        addChild($$, $3);
    }
    ;

// 表达式 - 保留布尔字面量
expression:
    TOK_ID
    {
        $$ = createNode(NODE_IDENTIFIER, $1);
    }
    | TOK_INT
    {
        $$ = createNode(NODE_CONSTANT, $1);
    }
    | TOK_FLOAT
    {
        $$ = createNode(NODE_CONSTANT, $1);
    }
    | TOK_TRUE
    {
        $$ = createNode(NODE_CONSTANT, "true");
    }
    | TOK_FALSE
    {
        $$ = createNode(NODE_CONSTANT, "false");
    }
    | TOK_LY expression TOK_RY
    {
        $$ = $2;  // 括号表达式：直接返回内部表达式
    }
    | expression TOK_ADD expression
    {
        $$ = createNode(NODE_OPERATOR, "+");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_SUB expression
    {
        $$ = createNode(NODE_OPERATOR, "-");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_MUL expression
    {
        $$ = createNode(NODE_OPERATOR, "*");
        addChild($$, $1);
        addChild($$, $3);
    }
    | expression TOK_DIV expression
    {
        $$ = createNode(NODE_OPERATOR, "/");
        addChild($$, $1);
        addChild($$, $3);
    }
    ;

与词法分析的联动

与 Flex 词法分析器的联动关键点

  1. yylex()：由 Flex 生成，Bison 的 yyparse() 会调用它获取下一个 token。
  2. yylval：Bison 生成的 %union 定义了 yylval 的类型，Flex 动作中对 yylval.string_value 赋值，Bison 产生式中通过 $n 访问。
  3. TOK_ 宏：Bison 生成 syntax.tab.h，其中每个 %token 被 #define 为整数。Flex 包含该头文件，在动作中返回这些宏。
  4. yyerror：Bison 在遇到语法错误时会调用该函数，打印错误信息。

语法和词法是最基础的部分，以一个具体示例作结：

源代码：hang { int a = 1+2; }

Token 流（Flex 返回）：
TOK_HANG TOK_LH TOK_INT_TYPE TOK_ID(“a”) TOK_ASSIGN TOK_INT(“1”) TOK_ADD TOK_INT(“2”) TOK_FEN TOK_RH

语法分析归约过程（简化）：

  1. TOK_INT(“1”) → expression → NODE_CONSTANT(“1”)
  2. TOK_INT(“2”) → expression → NODE_CONSTANT(“2”)
  3. expression “+” expression → expression → NODE_OPERATOR(“+”) 挂载两个常量节点
  4. TOK_INT_TYPE → type_specifier → NODE_TYPE(“int”)
  5. TOK_ID(“a”) TOK_ASSIGN expression → declaration_statement (decl_init) → NODE_DECLARATION(“decl_init”)，子节点为 type, identifier, 表达式节点
  6. statements → 链表包含上述 declaration
  7. TOK_HANG TOK_LH statements TOK_RH → program → NODE_PROGRAM，子节点为 statements 链表
  8. 根节点存入 ast_root

最终 AST 可用于语义分析和代码生成。

与语义分析部分的联动

这里预先定义了抽象语法树AST的实现，每规约一条产生式，就调用对应的C代码创建节点、连接父子或者兄弟关系。之后只要遍历AST并取出对应的值加以操作，就能完成语义分析和中间代码的生成（还可以附带行号到AST节点，语义分析报错的同时实现行号追踪，方便确定问题）。编译完成后记得销毁。

与全局符号表的联动

核心填充与检查动，这是与符号表交互最密集、最关键的阶段。

  • 填充信息：根据声明语句，将类型、作用域、维度（数组）、参数列表（函数）等属性填入对应条目。

  • 类型检查：在表达式或赋值语句中，查询操作符两边的标识符类型，判断是否兼容。

  • 重复声明检查：在同一作用域内插入同名标识符时，会触发错误。

  • 作用域管理：通过栈或树结构实现，支持标识符的屏蔽规则。

3.语义分析并生成中间代码

理论

==词法分析确定了每个词是合法的，语法分析确定了某些词组成的句子是合法的，语义分析则要确定在当前上下文中，这些合法句子是否仍然是有意义的，并生成中间代码==。

核心是如何在前两步记录下节点属性，并通过树遍历法等方式记录、传递这些属性。

实现（内含代码，不再单独摘出）

前两步已经生成了抽象语法树AST，接下来的操作都基于AST的遍历，具体如下：

语义分析的实现

语义分析负责：符号表管理、作用域处理、类型检查、变量声明/引用验证、未初始化警告等。核心代码位于 semantic.c 和 symbol.c。

① 符号表结构（common.h）
typedef struct SymbolEntry {
    char name[50];

    DataType data_type;       // int/float/bool/void
    int is_initialized;
    int scope_level;
    int memory_address;
    int line_declared;

} SymbolEntry;

typedef struct {
    SymbolEntry* buckets[100]; // 哈希桶

    int next_address;

符号表提供 insert_symbol、lookup_symbol、enter_scope、exit_scope 等接口（实现在 symbol.c 中）。

② 语义分析主函数 semantic_check（semantic.c）

该函数递归遍历 AST，根据节点类型执行不同动作。关键代码片段：

void semantic_check(ASTNode* node) {
    if (!node) return;
    switch (node->type) {
        case NODE_PROGRAM:
            for (ASTNode* child = node->first_child; child; child = child->next_sibling)
                semantic_check(child);
            break;
        case NODE_DECLARATION: {
            ASTNode* type_node = node->first_child;
            ASTNode* id_node = type_node->next_sibling;
            DataType var_type = string_to_type(type_node->value);
            // 遍历标识符链表
            for (ASTNode* current = id_node; current; current = current->next_sibling) {
                SymbolEntry* entry = insert_symbol(
                    current->value, SYM_VARIABLE, var_type, current_line);
                if (current->first_child) { // 有初始化表达式
                    entry->is_initialized = 1;
                    semantic_check(current->first_child);
                }
            }
            break;
        }
        case NODE_ASSIGNMENT: {
            ASTNode* left = node->first_child;
            ASTNode* right = left->next_sibling;
            SymbolEntry* entry = lookup_symbol(left->value);
            if (!entry) error("变量未声明");
            else entry->is_initialized = 1;
            semantic_check(right);
            break;
        }
        case NODE_IDENTIFIER: {
            SymbolEntry* entry = lookup_symbol(node->value);
            if (!entry) error("变量未声明");
            else if (!entry->is_initialized) warning("可能未初始化");
            break;
        }
        case NODE_WHEN: {
            ASTNode* cond = node->first_child;
            ASTNode* body = cond->next_sibling;
            semantic_check(cond);
            enter_scope();          // 进入新作用域
            semantic_check(body);
            exit_scope();
            break;
        }
        // 其他节点：INPUT、OUTPUT、OPERATOR 等继续递归
    }
}
③语义分析的调用时机

在 main() 中（syntax.y 末尾或单独的 main.c）：

③核心递归函数 generateCode

该函数同样遍历 AST，根据节点类型产生四元式。（同样位于quad.c）：

void generateCode(ASTNode* node) {
    if (!node) return;
    switch (node->type) {
        case NODE_PROGRAM:
        case NODE_STATEMENT_LIST:
            for (ASTNode* child = node->first_child; child; child = child->next_sibling)
                generateCode(child);
            break;
        case NODE_ASSIGNMENT: {
            ASTNode* left = node->first_child;      // 标识符
            ASTNode* right = left->next_sibling;    // 表达式
            char* right_val = generateExpression(right);
            if (right_val) {
                addQuadruple("=", right_val, "_", left->value);
                free(right_val);
            }
            break;
        }
        case NODE_INPUT:
            addQuadruple("IN", "_", "_", node->first_child->value);
            break;
        case NODE_OUTPUT: {
            char* expr_val = generateExpression(node->first_child);
            if (expr_val) {
                addQuadruple("OUT", expr_val, "_", "_");
                free(expr_val);
            }
            break;
        }
        case NODE_WHEN: {
            ASTNode* cond = node->first_child;
            ASTNode* body = cond->next_sibling;
            char* cond_val = generateExpression(cond);
            char* false_label = newLabel();
            addQuadruple("JZ", cond_val, "_", false_label);
            generateCode(body);
            addQuadruple("LABEL", false_label, "_", "_");
            free(cond_val);
            break;
        }
        // 其他节点...
    }
}
④表达式处理 generateExpression

递归生成表达式运算的四元式，并返回结果存放的临时变量名。

char* generateExpression(ASTNode* node) {
    if (!node) return NULL;
    switch (node->type) {
        case NODE_IDENTIFIER:
        case NODE_CONSTANT:
            return strdup(node->value);   // 直接返回变量名或常量字符串
        case NODE_OPERATOR: {
            ASTNode* left = node->first_child;
            ASTNode* right = left->next_sibling;
            char* left_val = generateExpression(left);
            char* right_val = generateExpression(right);
            // 常量折叠：如果左右都是数字，直接计算
            if (isStringNumeric(left_val) && isStringNumeric(right_val)) {
                int result = compute(left_val, node->value, right_val);
                char* temp = newTemp();
                char resultStr[32];
                sprintf(resultStr, "%d", result);
                addQuadruple("=", resultStr, "_", temp);
                free(left_val); free(right_val);
                return temp;
            }
            // 正常生成运算四元式
            char* temp = newTemp();
            addQuadruple(node->value, left_val, right_val, temp);
            free(left_val); free(right_val);
            return temp;
        }
        default: return NULL;
    }
}
⑤辅助函数
  • newTemp()：返回 t0, t1, … 格式的临时变量名。
  • newLabel()：返回 L0, L1, … 格式的标签名。
  • addQuadruple(op, arg1, arg2, result)：将四元式存入全局数组。

4.中间代码优化

理论

可以分为基本块内的局部优化，比如合并已知量、删除公共子表达式、删除无用赋值等；以及跨基本块的循环优化，比如代码外提（主要是将不变运算移到循环外）、强度削弱——用快操作替换慢操作（如乘转加）以及删除替换循环中的辅助变量等。

实现

出于暂时不支持通过when（这里的when不是while，相当于if）等实现循环，所以这里只进行局部优化；具体而言是只做了简单的常量折叠，代码如下：

该函数负责递归生成表达式运算的四元式，并返回结果存放的临时变量名。同时进行常量折叠（在生成阶段直接计算常量表达式）。

优化与输出

在 optimize.c 中，optimize_quadruples() 会遍历四元式数组，执行常量传播和常量折叠（与生成阶段的折叠互补）。最终通过 printQuadruples() 打印所有四元式。

5.目标代码生成（可结合nasm等生成可执行文件）

理论

通过优化后的四元式，生成最终的汇编等机器语言。核心在于分配调用寄存器，可以在局部性原理的基础上，计算活跃和待用信息，进而为分配/释放寄存器提供关键依据。

实现

codegen.c 中的 generateAssembly() / generateAssemblyToFile() 遍历四元式数组，将其映射为 x86-64 汇编指令。

四元式到汇编的转换核心逻辑

在 codegen.c 中，generateAssembly() 函数遍历全局四元式数组 quadruples[]，根据每个四元式的操作符 op 生成对应的 x86-64 汇编代码。

void generateAssembly() {
    FILE* asmFile = fopen("output.asm", "w");
    // ... 输出数据段、变量声明等 ...
    
    fprintf(asmFile, "section .text\n");
    fprintf(asmFile, "global _start\n");
    
    for (int i = 0; i < quadCount; i++) {
        if (strcmp(quadruples[i].op, "IN") == 0) {
            // 输入操作：调用 scanf 或系统调用
            fprintf(asmFile, "    push %s\n", quadruples[i].result);
            fprintf(asmFile, "    push input_format\n");
            fprintf(asmFile, "    call scanf\n");
            fprintf(asmFile, "    add esp, 8\n");
        }
        else if (strcmp(quadruples[i].op, "=") == 0) {
            if (isdigit(quadruples[i].arg1[0])) {
                // 常量赋值：mov dword [变量], 常量
                fprintf(asmFile, "    mov dword [%s], %s\n", 
                        quadruples[i].result, quadruples[i].arg1);
            } else {
                // 变量赋值：mov eax, [源] → mov [目标], eax
                fprintf(asmFile, "    mov eax, [%s]\n", quadruples[i].arg1);
                fprintf(asmFile, "    mov [%s], eax\n", quadruples[i].result);
            }
        }
        else if (strcmp(quadruples[i].op, "+") == 0) {
            // 加法：mov eax, [左] → add eax, [右] → mov [结果], eax
            fprintf(asmFile, "    mov eax, [%s]\n", quadruples[i].arg1);
            if (isdigit(quadruples[i].arg2[0]))
                fprintf(asmFile, "    add eax, %s\n", quadruples[i].arg2);
            else
                fprintf(asmFile, "    add eax, [%s]\n", quadruples[i].arg2);
            fprintf(asmFile, "    mov [%s], eax\n", quadruples[i].result);
        }
        else if (strcmp(quadruples[i].op, "*") == 0) {
            // 乘法：imul eax, [右]
            fprintf(asmFile, "    mov eax, [%s]\n", quadruples[i].arg1);
            if (isdigit(quadruples[i].arg2[0]))
                fprintf(asmFile, "    imul eax, %s\n", quadruples[i].arg2);
            else
                fprintf(asmFile, "    imul eax, [%s]\n", quadruples[i].arg2);
            fprintf(asmFile, "    mov [%s], eax\n", quadruples[i].result);
        }
        else if (strcmp(quadruples[i].op, "JZ") == 0) {
            // 条件跳转：cmp dword [条件], 0 → je 标签
            fprintf(asmFile, "    cmp dword [%s], 0\n", quadruples[i].arg1);
            fprintf(asmFile, "    je %s\n", quadruples[i].result);
        }
        else if (strcmp(quadruples[i].op, "LABEL") == 0) {
            // 标签定义
            fprintf(asmFile, "%s:\n", quadruples[i].arg1);
        }
        else if (strcmp(quadruples[i].op, "OUT") == 0) {
            // 输出：push dword [值] → push output_format → call printf
            fprintf(asmFile, "    push dword [%s]\n", quadruples[i].arg1);
            fprintf(asmFile, "    push output_format\n");
            fprintf(asmFile, "    call printf\n");
            fprintf(asmFile, "    add esp, 8\n");
        }
    }
    // ... 程序退出代码 ...
}

映射规则总结

四元式操作 汇编指令模式
= 3 b mov dword [b], 3
= a b mov eax, [a] → mov [b], eax
+ a b t0 mov eax, [a] → add eax, [b] → mov [t0], eax
* a b t1 mov eax, [a] → imul eax, [b] → mov [t1], eax
IN _ _ a push a → push input_format → call scanf
OUT a _ _ push dword [a] → push output_format → call printf
JZ cond L0 cmp dword [cond], 0 → je L0
LABEL L0 _ _ L0:

完整示例：a = b + c * 2 的翻译过程

①源代码（Hang 语言）

hang {
    int a, b, c;
    b = 3;
    c = 4;
    a = b + c * 2;
    out(a);
}

②AST 构建（语法分析阶段）

对应的 AST 结构（简化）：

NODE_PROGRAM
  ├─ NODE_DECLARATION ("decl") → [int] [a, b, c]
  ├─ NODE_ASSIGNMENT ("=") → [b] [3]
  ├─ NODE_ASSIGNMENT ("=") → [c] [4]
  └─ NODE_ASSIGNMENT ("=") → [a] 
       └─ NODE_OPERATOR ("+")
            ├─ NODE_IDENTIFIER ("b")
            └─ NODE_OPERATOR ("*")
                 ├─ NODE_IDENTIFIER ("c")
                 └─ NODE_CONSTANT ("2")

③四元式生成（quad.c 中 generateCode 遍历 AST 产生）

遍历过程的关键步骤：

步骤 处理的 AST 节点 动作 生成的四元式
1 NODE_ASSIGNMENT (b = 3) 常量直接赋值 = 3 _ b
2 NODE_ASSIGNMENT (c = 4) 常量直接赋值 = 4 _ c
3 NODE_OPERATOR ("*") 递归生成 c * 2 t0 = c * 2
4 NODE_OPERATOR ("+") 递归生成 b + t0 t1 = b + t0
5 NODE_ASSIGNMENT (a = ...) 将结果赋给 a = t1 _ a

最终四元式序列：

序号  操作符  操作数1  操作数2  结果
0    =       3       _       b
1    =       4       _       c
2    *       c       2       t0
3    +       b       t0      t1
4    =       t1      _       a
5    OUT     a       _       _

④汇编代码生成（codegen.c 遍历四元式）

对每个四元式执行 generateAssembly() 中的映射规则：

四元式 0: = 3 _ b

mov dword [b], 3

四元式 1: = 4 _ c

mov dword [c], 4

四元式 2: * c 2 t0

mov eax, [c]       ; 将 c 的值加载到 eax
imul eax, 2        ; eax = eax * 2
mov [t0], eax      ; 结果存入临时变量 t0

四元式 3: + b t0 t1

mov eax, [b]       ; 加载 b 的值
add eax, [t0]      ; eax = eax + t0
mov [t1], eax      ; 结果存入临时变量 t1

四元式 4: = t1 _ a

mov eax, [t1]      ; 加载 t1 的值
mov [a], eax       ; 存入变量 a

四元式 5: OUT a _ _

push dword [a]     ; 将 a 的值压栈
push output_format ; 压入格式字符串 "%d\n"
call printf        ; 调用 printf
add esp, 8         ; 清理栈

==完整汇编输出==

section .data
    a dd 0
    b dd 0
    c dd 0
    t0 dd 0
    t1 dd 0
    input_format db "%d", 0
    output_format db "%d", 10, 0

section .text
    global _start
    extern printf, scanf

_start:
    ; = 3 _ b
    mov dword [b], 3
    ; = 4 _ c
    mov dword [c], 4
    ; * c 2 t0
    mov eax, [c]
    imul eax, 2
    mov [t0], eax
    ; + b t0 t1
    mov eax, [b]
    add eax, [t0]
    mov [t1], eax
    ; = t1 _ a
    mov eax, [t1]
    mov [a], eax
    ; OUT a
    push dword [a]
    push output_format
    call printf
    add esp, 8
    ; 退出
    mov eax, 60
    xor edi, edi
    syscall

总结

阶段 核心函数 输入 → 输出
语法分析 → AST Bison 规则 源代码 → AST
语义分析 semantic_check AST + 符号表（变量检查、作用域）
四元式生成 generateCode + generateExpression AST → 四元式数组
汇编生成 generateAssembly 四元式数组 → x86-64 汇编代码

通过这个示例，可以清晰地看到：每个四元式操作符都有一条或多条对应的汇编指令模板，generateAssembly() 通过 strcmp 识别操作符，然后按照模板输出汇编代码，最终生成完整的可汇编文件。

语义分析与四元式转化的关系

阶段 输入 输出 依赖
语义分析 AST（ast_root） 填充的符号表、错误/警告信息 仅依赖 AST 结构
四元式生成 AST（ast_root） 四元式数组 quadruples[] 依赖 AST 结构，并可读取符号表（例如检查变量是否存在）

虽然四元式生成也在遍历 AST，但它不负责语义检查（此时语义已经分析完毕）。实际调用顺序为：

yyparse();               // 生成 ast_root
semantic_check(ast_root);   // 语义分析
generateQuadruples(ast_root); // 四元式生成
optimize_quadruples();       // 优化
generateAssembly();          // 汇编生成
  • 语义分析：通过 semantic_check 递归遍历 AST，利用符号表完成声明检查、作用域管理、变量引用验证、类型检查等。代码在 semantic.c + symbol.c。
  • 四元式转化：通过 generateCode 递归遍历 AST，为每个语法结构生成对应的四元式（=、+、IN、OUT、JZ 等），表达式生成临时变量并支持常量折叠。代码在 quad.c。
  • 两者分离：语义分析先于四元式生成，通过全局 ast_root 和符号表传递信息，无直接耦合。

（6.生成可执行文件）

汇编文件本身不是可执行文件，还需要两个步骤：汇编和链接：

.asm 文件  →  [汇编器] →  .o 文件  →  [链接器] →  可执行文件
(汇编代码)    (nasm)    (目标文件)    (ld)       (ELF)

①汇编（Assemble）：将汇编代码转换为机器码

使用 NASM（Netwide Assembler）：

nasm -f elf64 test.asm -o test.o
  • -f elf64：指定输出格式为 64 位 ELF 目标文件（Linux 标准格式）
  • test.o：目标文件，包含机器指令，但地址尚未重定位

②链接（Link）：将目标文件转换为可执行文件

使用 LD（GNU Linker）：

ld test.o -o test
  • 重定位地址，解析外部符号（如 printf、scanf 虽然本例没有调用 libc 的实际需求，但代码中调用了这些函数，可能需要链接 C 库，更完整的命令是 ld -dynamic-linker /lib64/ld-linux-x86-64.so.2 -lc test.o -o test）
  • 设置程序入口点（_start 或 main）
  • 生成 ELF 可执行文件

# 2. 运行编译器生成汇编
./compiler $1

# 3. 获取源文件名（不含扩展名）
filename=$(basename "$1" .hang)

# 4. 汇编
nasm -f elf64 ${filename}.asm -o ${filename}.o

# 5. 链接（如果需要 C 库，会加上 -lc）
ld ${filename}.o -o ${filename}

# 6. 运行
./${filename}

一键执行：

chmod +x build.sh
./build.sh test.hang

   ↓

   ↓
